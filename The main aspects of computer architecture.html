The main aspects of computer architecture

  1. Building an Instruction Set Architecture ISA is an interface between computer software and hardware, and it can also be seen as the viewpoint of the device programmer. Computers do not understand high-level languages that have a few, if any, language elements that translate directly into the mother machine's operation codes.

    The therapist only understands the instructions encoded in some numerical fashion, usually binary numbers. Software tools, such as compilers, translate high-level languages, such as C, into instructions. In addition to instructions, ISA determines which components are available on your computer for a program such as data types and records, status manipulation, and memory.
  2. Computer Organization The Computer Organization helps improve performance-based products. For example, software engineers need to know the processing power of processors.

    They may need to improve software in order to get the most performance at the lowest cost. This could require a very detailed analysis of the computer organization.

    For example, in multimedia decoding, designers may need to arrange for most of the data being processed in the data path faster.
  3. The computer organization also helps plan the processor selection for a specific project. Multimedia projects may require very fast data access, while the supervisory program may need fast interruptions. Sometimes some tasks need .

    additional components as well. For example, a computer that is capable of virtualization needs virtual memory devices so that memory for different simulated computers can still be separated. Computer organization and features also affects power consumption and processor cost.
